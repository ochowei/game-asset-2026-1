# Path B v1.4.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship Procforge v1.4.0: medieval-fantasy theme moves from hand-coded primitives to AI-designed SVG bases varied at runtime by procgen (palette + light transform + decoration). Other 3 themes unchanged. New brand pillars: Deterministic Runtime / Procedural Variation / Open-source Generator.

**Architecture:** Authoring-time AI generates 5 base SVGs per subject (30 total for medieval-fantasy's 6 subjects). Bases live as `*.svg` files in `_assets/`, converted to TS string-array modules at build time. Runtime primitive picks one base via seeded RNG, replaces palette `{{role}}` tokens with palette colors, applies a light transform (rotate ±8°, scale 0.95–1.05, optional mirror), and returns the SVG fragment. Determinism contract `(theme, seed) → byte-identical` preserved.

**Tech Stack:** TypeScript 5.6, pnpm 9 monorepo, Vitest 2, tsup 8, Node 22, seedrandom, @resvg/resvg-js, jszip.

**Spec:** `docs/superpowers/specs/2026-05-06-path-b-v1.4.0-design.md` (commit `ce83035`).

Create `packages/core/src/primitives/base-variation.ts`:

```typescript
import { pick, range, type RNG } from '../seed';
import { pickColor, type Palette, type ColorRole } from '../palette';
import { round2 } from '../numeric';
import { svgElement } from '../svg-emitter';

export interface ApplyBaseVariationOptions {
  rng: RNG;
  bases: readonly string[];
  palette: Palette;
  size: number;
  allowMirror: boolean;
}

const ROLES: readonly ColorRole[] = ['primary', 'secondary', 'accent', 'neutral'];
const BASE_VIEW = 64;

export function applyBaseVariation(opts: ApplyBaseVariationOptions): string {
  if (opts.bases.length === 0) throw new Error('applyBaseVariation: bases empty');

  // Step 1: pick base
  const base = pick(opts.rng, opts.bases);

  // Steps 2-5: pick palette color per role (always 4 draws regardless of usage)
  let body = base;
  for (const role of ROLES) {
    const hex = pickColor(opts.rng, opts.palette, role);
    body = body.split(`{{${role}}}`).join(hex);
  }

  // Step 6: rotation
  const rotateDeg = range(opts.rng, -8, 8);
  // Step 7: uniform scale
  const scaleVar = range(opts.rng, 0.95, 1.05);
  // Step 8: conditional mirror
  const mirror = opts.allowMirror ? opts.rng() < 0.5 : false;

  const canvasScale = opts.size / BASE_VIEW;
  const sx = (mirror ? -1 : 1) * scaleVar * canvasScale;
  const sy = scaleVar * canvasScale;

  // transform reads right-to-left:
  // 1) translate(-32,-32)  → centre base at origin
  // 2) rotate(rotateDeg)
  // 3) scale(sx, sy)
  // 4) translate(size/2, size/2) → place at canvas centre
  const tx = round2(opts.size / 2);
  const ty = round2(opts.size / 2);
  const transform =
    `translate(${tx} ${ty}) ` +
    `scale(${round2(sx)},${round2(sy)}) ` +
    `rotate(${round2(rotateDeg)}) ` +
    `translate(-${round2(BASE_VIEW / 2)} -${round2(BASE_VIEW / 2)})`;

  return svgElement('g', { transform }, body);
}
```

```typescript
import { readdirSync, readFileSync, writeFileSync, existsSync } from 'node:fs';
import { join, basename } from 'node:path';

interface BaseGroup {
  subject: string;
  files: { path: string; content: string; index: number }[];
}

const THEME_ROOT = 'packages/themes/medieval-fantasy/src/subjects';
const ASSETS_DIR = join(THEME_ROOT, '_assets');

const ROLE_TOKENS = ['{{primary}}', '{{secondary}}', '{{accent}}', '{{neutral}}'];

function validate(filename: string, svg: string): void {
  if (!/^[\x00-\x7F]*$/.test(svg)) throw new Error(`${filename}: non-ASCII characters`);
  if (!svg.includes('viewBox="0 0 64 64"')) throw new Error(`${filename}: missing viewBox="0 0 64 64"`);
  if (svg.includes('<image')) throw new Error(`${filename}: contains <image>`);
  if (svg.includes('<script')) throw new Error(`${filename}: contains <script>`);
  if (svg.includes('<style')) throw new Error(`${filename}: contains <style>`);
  if (svg.includes('xlink:href')) throw new Error(`${filename}: contains external xlink:href`);
  // require at least one palette token; warn (not fail) if none
  const hasToken = ROLE_TOKENS.some((t) => svg.includes(t));
  if (!hasToken) console.warn(`${filename}: no palette tokens — base will render with no palette substitution`);
}

function extractBody(svg: string): string {
  // Strip <svg ...> wrapper — keep contents as-is.
  const open = svg.match(/<svg[^>]*>/);
  if (!open) throw new Error('no <svg> open tag');
  const close = svg.lastIndexOf('</svg>');
  if (close === -1) throw new Error('no </svg> close tag');
  const body = svg.slice(open.index! + open[0].length, close).trim();
  if (!body.startsWith('<g')) throw new Error('body does not start with single <g> root');
  return body;
}

function escapeForTemplateLiteral(s: string): string {
  return s.replace(/\\/g, '\\\\').replace(/`/g, '\\`').replace(/\$\{/g, '\\${');
}

function groupFiles(): BaseGroup[] {
  if (!existsSync(ASSETS_DIR)) {
    console.log(`No _assets dir at ${ASSETS_DIR} — skipping`);
    return [];
  }
  const files = readdirSync(ASSETS_DIR).filter((f) => f.endsWith('.svg')).sort();
  const groups = new Map<string, BaseGroup>();
  for (const f of files) {
    const m = f.match(/^(.+)-(\d+)\.svg$/);
    if (!m) throw new Error(`${f}: filename must be <subject>-<index>.svg`);
    const subject = m[1] as string;
    const idx = Number(m[2]);
    const content = readFileSync(join(ASSETS_DIR, f), 'utf8');
    validate(f, content);
    const body = extractBody(content);
    if (!groups.has(subject)) groups.set(subject, { subject, files: [] });
    groups.get(subject)!.files.push({ path: f, content: body, index: idx });
  }
  return [...groups.values()].map((g) => ({
    subject: g.subject,
    files: g.files.sort((a, b) => a.index - b.index),
  }));
}

function writeModule(group: BaseGroup): void {
  const exportName = group.subject.replace(/-([a-z])/g, (_, c: string) => c.toUpperCase()) + 'Bases';
  const lines: string[] = [];
  lines.push('// AUTO-GENERATED by scripts/build-bases.ts. Do not edit by hand.');
  lines.push(`// Source: packages/themes/medieval-fantasy/src/subjects/_assets/${group.subject}-*.svg`);
  lines.push('');
  lines.push(`export const ${exportName}: readonly string[] = [`);
  for (const f of group.files) {
    lines.push(`  \`${escapeForTemplateLiteral(f.content)}\`,`);
  }
  lines.push('] as const;');
  lines.push('');
  const out = join(THEME_ROOT, `${group.subject}.bases.ts`);
  writeFileSync(out, lines.join('\n'), 'utf8');
  console.log(`wrote ${out} (${group.files.length} bases)`);
}

function main(): void {
  const groups = groupFiles();
  if (groups.length === 0) {
    console.log('No base groups found. Exiting.');
    return;
  }
  for (const g of groups) writeModule(g);
  console.log(`build-bases: ${groups.length} subject groups, ${groups.reduce((n, g) => n + g.files.length, 0)} bases total.`);
}

main();
```

```typescript
// Render candidate SVG bases at multiple sizes for human curation.
// Usage: pnpm tsx scripts/qa-base-preview.ts <input-dir>
// Output: <input-dir>/_preview/<subject>-N-<size>.png + <subject>-sheet.html

import { readdirSync, readFileSync, writeFileSync, mkdirSync, existsSync } from 'node:fs';
import { join, basename } from 'node:path';
import { renderSvgToPng } from '@procforge/cli/render';

const SIZES = [16, 32, 64, 128];
const STAND_IN_PALETTE: Record<string, string> = {
  '{{primary}}': '#a85432',
  '{{secondary}}': '#3a5a4a',
  '{{accent}}': '#d4a73a',
  '{{neutral}}': '#1f1a14',
};

async function main(): Promise<void> {
  const inputDir = process.argv[2];
  if (!inputDir) {
    console.error('Usage: pnpm tsx scripts/qa-base-preview.ts <input-dir>');
    process.exit(1);
  }
  if (!existsSync(inputDir)) {
    console.error(`input-dir does not exist: ${inputDir}`);
    process.exit(1);
  }
  const previewDir = join(inputDir, '_preview');
  mkdirSync(previewDir, { recursive: true });

  const svgs = readdirSync(inputDir).filter((f) => f.endsWith('.svg')).sort();
  const cells: string[] = [];
  for (const f of svgs) {
    let content = readFileSync(join(inputDir, f), 'utf8');
    for (const [tok, hex] of Object.entries(STAND_IN_PALETTE)) {
      content = content.split(tok).join(hex);
    }
    for (const size of SIZES) {
      const buf = await renderSvgToPng(content, size);
      writeFileSync(join(previewDir, `${basename(f, '.svg')}-${size}.png`), buf);
    }
    const stem = basename(f, '.svg');
    cells.push(
      `<figure><figcaption>${stem}</figcaption>` +
        SIZES.map((s) => `<img src="${stem}-${s}.png" width="${s}" height="${s}" alt="${stem}@${s}">`).join('') +
        `</figure>`,
    );
  }

  const html = `<!doctype html><meta charset="utf-8"><title>QA preview</title>
<style>
body{margin:16px;background:#1a1a2e;color:#eee;font-family:system-ui,sans-serif}
figure{display:inline-block;margin:6px;padding:6px;background:#0f1020;border-radius:6px;text-align:center}
figcaption{font-size:11px;margin-bottom:4px;color:#aaa}
img{background:#fff;margin:2px;image-rendering:pixelated}
</style>
${cells.join('\n')}`;
  writeFileSync(join(previewDir, 'index.html'), html, 'utf8');
  console.log(`Wrote ${svgs.length} candidate previews to ${previewDir}/`);
  console.log(`Open ${join(previewDir, 'index.html')} in a browser to curate.`);
}

main().catch((e) => {
  console.error(e);
  process.exit(1);
});
```

```markdown
# Procforge Path B base SVG prompt

> Committed for transparency per `docs/AI-AUTHORING.md`. Substitute the placeholders before sending to the LLM.

You are designing a single icon SVG for the Procforge open-source procedural icon generator. Your output becomes one of 5 base SVGs for the named subject; downstream procgen will recolour it via palette substitution and apply small whole-image transforms (rotate ±8°, uniform scale ±5%). Subject silhouette is what matters. Surface texture and gradients are NOT used (no `<defs>`, no gradients).

## Required output format

Return ONLY a valid `<svg>` document, no commentary, no markdown. The SVG MUST satisfy ALL of:

1. Root: `<svg xmlns="http://www.w3.org/2000/svg" width="64" height="64" viewBox="0 0 64 64">`
2. Single `<g>` element wraps all geometry inside the `<svg>`.
3. Subject silhouette stays inside the inner box `4..60` on both axes (~10% padding).
4. Palette colours expressed ONLY as one of these tokens:
   - `{{primary}}` — main subject body (largest fill area)
   - `{{secondary}}` — secondary surfaces (mid-area accents)
   - `{{accent}}` — small highlight details (tiny fills)
   - `{{neutral}}` — strokes / dark contrast
   You may use a subset (e.g. only `{{primary}}` + `{{neutral}}`) if appropriate.
5. Tokens appear in `fill="{{role}}"` and `stroke="{{role}}"` attributes only.
6. Stroke widths inline as `stroke-width` numeric attributes (no calc, no CSS).
7. NO `<defs>`, NO `<style>`, NO `<image>`, NO `<script>`, NO external refs (no `xlink:href`).
8. ASCII characters only.
9. Coordinates align to the integer/half-integer grid where reasonable.

## Subject

**Name:** `{SUBJECT_NAME}`
**Description:** `{SUBJECT_DESCRIPTION}`
**Theme:** medieval-fantasy. Visual register: clean line-icon style (think Lucide / Phosphor / Tabler), readable as the named subject at 32 px. NOT pixel art, NOT highly stylised — a clear silhouette.
**Style notes:** thicker strokes than typical (stroke-width 2–4 at 64 viewBox), mild fantasy character, no overly literal realism. The icon must read instantly as "{SUBJECT_NAME}" to a non-author reviewer.

## Variation

Produce **N=5 visually distinct silhouettes** for the same subject, each as a separate SVG document. Vary the silhouette dimensions, posture, or stylistic emphasis (e.g. for sword: long thin blade vs broad blade vs hilt-emphasised). Do NOT vary just colours — colour is supplied by the runtime palette layer.

Output the 5 SVGs separated by a single blank line. No other text.
```

```markdown
# AI Authoring Pipeline

> Status: introduced in v1.4.0 for the medieval-fantasy theme. Other themes remain on the v1.3.1 hand-coded primitive model documented in `THEME-AUTHORING.md`.

## What this is

Procforge's authoring pipeline allows base SVGs to be generated by an LLM under human curation. The runtime pipeline that buyers run is unaffected: it remains 100% deterministic, offline, and AI-free.

```
AUTHORING TIME (one-time, by author)        RUNTIME (in buyer's machine)
────────────────────────────────────        ──────────────────────────────
LLM generates SVG candidates                generateOne({theme, seed, size})
   ↓                                            ↓ deterministic RNG
author curates (accept/reject/regen)            ↓ pick base (RNG)
   ↓                                            ↓ palette substitution
SVG committed → build:bases → *.bases.ts        ↓ light transform (RNG)
   ↓                                            ↓ emit final SVG fragment
pnpm test + visual QA                           ↓ no AI involvement
```

## When to use

Use this pipeline when authoring a new theme (or converting an existing theme) if you have access to an LLM. The hand-coded model is still valid; pick whichever fits your workflow.

## End-to-end loop

1. **Set up scratch dir.** `mkdir -p /tmp/path-b-candidates/<subject>`
2. **Run the prompt.** `scripts/ai-base-prompt.md` is the committed prompt template. Substitute `{SUBJECT_NAME}` and `{SUBJECT_DESCRIPTION}`, paste into your LLM, save returned SVGs as `cand-1.svg`, `cand-2.svg`, etc.
3. **Render previews.** `pnpm tsx scripts/qa-base-preview.ts /tmp/path-b-candidates/<subject>` writes PNGs at 16/32/64/128 to `_preview/` plus an `index.html` for browser inspection.
4. **Curate.** Apply the rubric below. Re-prompt as needed.
5. **Copy curated bases** into `packages/themes/<theme>/src/subjects/_assets/<subject>-{1..5}.svg`.
6. **Run** `pnpm build:bases` to regenerate `<subject>.bases.ts`.
7. **Run tests:** `pnpm --filter @procforge/theme-<theme> test`
8. **Commit** SVG + bases + (if newly converting a primitive) the rewritten `<subject>.ts` and test together.

## SVG conventions (non-negotiable)

Every committed base SVG must satisfy:

1. `viewBox="0 0 64 64"` exactly.
2. Subject silhouette stays inside the inner box `4..60` (~10% padding).
3. Single `<g>` element wraps all geometry inside the `<svg>`.
4. Palette colours expressed only as placeholder tokens:
   - `{{primary}}`, `{{secondary}}`, `{{accent}}`, `{{neutral}}`
5. Tokens appear in `fill="{{role}}"` and `stroke="{{role}}"` attributes.
6. Stroke widths inline as `stroke-width` numeric values.
7. No `<defs>`, `<style>`, `<image>`, `<script>`, no `xlink:href`.
8. ASCII characters only.

`scripts/build-bases.ts` validates these on build; `bases-validation.test.ts` re-validates in the test suite.

## Curation rubric (rejection conditions)

A candidate base is rejected if any of:

- Silhouette doesn't read as the named subject at 32 px to a non-author reviewer.
- A palette token is missing or wired to the wrong semantic role (e.g. `{{accent}}` used as the main body fill).
- Path data is messy: duplicate or zero-area subpaths, off-grid coordinates more than 0.5 unit from the integer/half-integer grid, attribute noise, unused wrapper `<g>`s.
- Visual similarity to another already-shipped base for the same subject is too high — the 5 bases must be visually distinct.
- Violates any SVG convention above.

If fewer than 5 candidates pass curation, re-prompt with explicit feedback (e.g. "previous candidates had blade clipping the viewBox; produce 3 more keeping silhouette inside 4..60 range").

## Per-subject mirror eligibility

Each primitive declares `allowMirror: boolean` when calling `applyBaseVariation`. Set to `false` for asymmetric subjects (sword, axe, scroll, anything with a directional handle). Set to `true` only when the subject is left-right symmetric (shield, gem, potion, ring).

## License posture

The repo is MIT. Base SVGs in `_assets/` are LLM-generated under human curation and are released under MIT alongside the rest of the codebase.

The MIT license grants downstream users whatever rights the licensor holds. In jurisdictions where AI-generated content is uncopyrightable (e.g. USCO 2023 guidance), the AI-derived portions are de facto in the public domain — which is more permissive than MIT and beneficial for downstream commercial use.

No AI is invoked at runtime. Buyers' generated icons depend only on Procforge code and the committed bases.

## Brand transparency

`scripts/ai-base-prompt.md` is committed to the repo. Anyone can read what was sent to the LLM. This is the "Open-source Generator" pillar applied to the authoring step.
```
